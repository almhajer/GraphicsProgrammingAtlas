# Runtime Brief

## الهدف
تشغيل سريع وخفيف لمهام التطوير داخل المشروع مع تقليل استهلاك السياق.

## القواعد السريعة
- عدّل المشروع الحالي فقط
- لا تغيّر الهوية البصرية الحالية
- لا تكسر القائمة الرئيسية أو التوزيع الحالي
- العربية هي اللغة الأساسية
- حافظ على الأداء الخفيف
- اختر مرحلة صغيرة واحدة فقط في كل مرة
- اقرأ `work-progress.json` أولًا
- اقرأ الملفات الموجودة في `read_next` فقط
- اقرأ المهارة المرتبطة بالمهمة الحالية فقط
- لا تفحص الملفات غير المرتبطة بالمرحلة
- في المهام التعليمية الجديدة: طبّق تلقائيًا المعنى الحقيقي، الشرح العميق، الأمثلة المتنوعة، وتعليقات الكود الهادفة
- حدّث checkpoint و `work-progress.json` بعد الانتهاء
