# قوالب البلوكات

هذا الملف يحدد البلوكات المرئية الأساسية التي ينبغي إعادة استخدامها بدل اختراع تنسيقات جديدة في كل صفحة.

## البلوكات الأساسية

- `page-header`
  ترويسة الصفحة الأساسية، وتحتوي العنوان والوصف وأحيانًا المعاينة أو الإجراءات الرئيسية.

- `hero`
  نسخة أوسع من الترويسة تستخدم لصفحات الأمثلة أو الصفحات المحورية ذات المعاينة المرئية.
  التطبيق الحالي: `renderExampleHeroSection(...)` داخل `js/modules/ui-blocks.js`.

- `info-section`
  البطاقة القياسية لعرض فكرة واحدة مستقلة: هدف، شرح، ملاحظات، متطلبات، عناصر مرتبطة.

- `content-card prose-card`
  بطاقة نصية للشرح الممتد، وتصلح للفقرات التعليمية أو التوضيحات المفهومية.

- `info-grid`
  شبكة بطاقات معلومات مختصرة مثل المتطلبات أو المقاييس أو نقاط الملخص.

- `example-section`
  حاوية مثال كامل عند الحاجة إلى ربط الشرح مع الكود أو الناتج.

- `code-block-card`
  بطاقة الكود الرسمية مع العنوان ووظائف النسخ والتمييز.

- `see-also-links`
  روابط مرجعية مرتبطة بعنصر أو مثال، وتظهر داخل بطاقة مستقلة.

- `library-example-preview-section`
  شريط أو شبكة استعراض للفهرسة البصرية للأمثلة في الصفحات الرئيسية للأقسام.

## قواعد الاستخدام

1. لا تضع `info-section` داخل `info-section` أخرى.
2. إذا كان المحتوى وحدة منطقية مستقلة، امنحه بطاقة مستقلة بدل دمجه مع ما بعده.
3. الصور والمعاينات يجب أن تتبع نفس نسبة العرض والحواف داخل نفس المجال.
4. عناوين البطاقات المهمة مثل `الهدف منه` و`مظلّل الرؤوس` و`مظلّل الأجزاء` تستخدم نفس الوزن اللوني والبصري.
5. روابط `see-also-links` لا تُترك عائمة خارج البطاقة؛ توضع دائمًا داخل حاوية مرئية منسقة.
6. إذا كان نفس `hero` أو نفس بطاقة الشرح يتكرر بين أكثر من مكتبة، يجب نقله إلى دالة render مشتركة بدل نسخ HTML مرة أخرى.
7. أي معاينة مرئية داخل البطاقة يجب أن تمر عبر stage داخلي يقيّد الحجم (`*-visual-stage` أو `*-hero-shot-stage`) حتى لا تتجاوز الحدود.

## ما هو موحّد الآن

- `renderExampleHeroSection(...)`
  القالب الرسمي لترويسات الأمثلة والعناصر التي تعرض معاينة على اليسار وبطاقة العنوان على اليمين.

- `renderExamplePageLayout(...)`
  قالب تركيب صفحات الأمثلة من `hero` + أقسام مستقلة، من دون إعادة كتابة `<section class="info-section">` في كل مسار.

- `renderExampleIndexLayout(...)`
  قالب موحد لفهارس الأمثلة التي تستخدم `page-header` ثم جسم الصفحة أسفلها.

- `renderSectionCollection(...)`
  مجمّع أقسام يسمح بتمرير أقسام مغلفة مسبقًا أو محتوى داخلي يُغلّف تلقائيًا في `info-section`.

- `renderUnifiedExampleOverviewCard(...)`
  بطاقة النظرة العامة المشتركة للمثال عندما يحتوي على هدف + معاينة.

- `renderUnifiedExampleSectionCard(...)`
  قالب البطاقة النصية القياسية للأقسام المنفصلة داخل صفحات الأمثلة.

- `renderUnifiedExampleSupportCard(...)`
  قالب بطاقات الدعم مثل النتيجة المتوقعة والعناصر المرتبطة والملاحظات السريعة.

- `renderLibraryExamplePreviewCard(...)`
  بطاقة المعاينة القياسية التي تعيد استخدامها فهارس أمثلة Vulkan وGLSL وSDL3 وDear ImGui وغيرها.

- `renderLibraryExamplePreviewSection(...)`
  الغلاف الموحد لقسم استعراض الأمثلة مع العنوان والوصف وزر الانتقال للفهرس الكامل.

- `*-hero-shot-stage`
  طبقة ضبط حجم المعاينات داخل ترويسات `hero`.

- `*-visual-stage`
  طبقة ضبط حجم المعاينات داخل بطاقات التمثيل البصري أو البطاقات الداخلية.

## نمط إنشاء قسم جديد

1. حدّد إن كان المحتوى: مرجع، مثال، درس، أم صفحة فهرسة.
2. اختر البلوكات من القائمة أعلاه قبل كتابة HTML.
3. ابدأ من الدوال المشتركة الموجودة فعلًا قبل كتابة قالب جديد.
4. استخدم أسماء CSS الموجودة أصلًا قبل إضافة صنف جديد.
5. إذا اضطررت إلى بلوك جديد، وثّقه هنا مباشرة مع وظيفته وحدود استخدامه.

## تقسيم مقترح مستقبلي

- `renderPageHeader(...)`
- `renderInfoSection(...)`
- `renderCodeBlockCard(...)`
- `renderRelatedLinksCard(...)`
- `renderExamplePreviewGrid(...)`
- `renderEntityInfoGrid(...)`

الهدف من هذا التقسيم أن تصبح البلوكات قوالب فعلية داخل JavaScript، لا مجرد أنماط CSS متشابهة. وقد بدأ هذا فعليًا الآن داخل `js/modules/ui-blocks.js` لمسار `hero` وبطاقات الأمثلة الموحدة وتجميع الأقسام.
