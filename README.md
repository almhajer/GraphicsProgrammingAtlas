# GraphicsProgrammingAtlas

مرجع عربي تقني لتعلّم وفهم أدوات ومكتبات البرمجة الرسومية والوسائط، مع تركيز على الشرح العملي والمعنى البرمجي الحقيقي بدل الاكتفاء بالترجمة الحرفية أو السرد السطحي.

## ما هو هذا المشروع؟

هذا المشروع عبارة عن موقع ويب ثابت (`static site`) يجمع بين:

- مرجع عربي منظّم للعناصر البرمجية
- شروحات تعليمية موجهة للمبتدئ لكن بعمق تقني حقيقي
- أمثلة كود عملية ومبسطة
- روابط داخلية بين المفاهيم والعناصر المتقاربة
- واجهة خفيفة وسريعة مناسبة للتصفح المباشر عبر المتصفح أو GitHub Pages

الهدف ليس نسخ التوثيق الرسمي كما هو، بل إعادة تقديمه بطريقة تساعد المتعلم العربي على فهم:

- ما هو العنصر البرمجي فعلًا
- لماذا يُستخدم
- متى يكون مناسبًا
- ما الفرق بينه وبين العناصر القريبة منه
- كيف يظهر في كود واقعي

## الهدف من الموقع

المشروع بُني ليكون قاعدة معرفية عربية متخصصة في البرمجة الرسومية والأنظمة المحيطة بها، بحيث يستطيع المطور الانتقال بين:

- المفاهيم التعليمية
- العناصر المرجعية
- الأمثلة
- التصنيفات
- النتائج المرتبطة

بشكل سريع ومنطقي داخل واجهة واحدة.

الهدف العملي من ذلك:

- تقليل الحاجز اللغوي أمام المبرمج العربي
- ربط التوثيق الرسمي بسياق تعليمي أوضح
- تقديم شروحات أصدق تقنيًا من مجرد الترجمة
- بناء مرجع قابل للتوسع تدريجيًا دون إعادة بناء المشروع من الصفر

## ما الذي يغطيه الموقع حاليًا؟

الموقع يجمع عدة مسارات رئيسية داخل نفس الواجهة، من أبرزها:

- `Vulkan`
- `C++`
- `CMake`
- `SDL3`
- `SDL3Audio`
- `FFmpeg`
- `GLSL`
- `Dear ImGui`
- الدروس التعليمية
- صفحات الملفات والفهارس المرتبطة

كل قسم قد يتضمن:

- فهارس مكتبات
- دوال
- أنواع وبنى
- تعدادات وثوابت وماكرو
- أمثلة
- صفحات شرح تفصيلي

## فلسفة المحتوى

هذا المشروع يعتمد على عدة مبادئ أساسية:

- العربية هي اللغة الأساسية للموقع
- الشرح يركز على المعنى الحقيقي والاستخدام العملي
- الأمثلة يجب أن تضيف فهمًا، لا مجرد زخرفة
- الربط بين العناصر مهم حتى لو لم يكن الربط الصريح موجودًا أصلًا
- الأداء الخفيف أولوية
- التطوير يتم على مراحل صغيرة وآمنة

## البنية العامة للمشروع

المشروع ليس تطبيقًا معقدًا يعتمد على خادم خلفي، بل موقع ثابت يعتمد على HTML وCSS وJavaScript وملفات JSON محلية.

أهم الأجزاء:

- [`index.html`](/Users/abdulkafi/Downloads/GraphicsProgrammingAtlas/index.html): نقطة الدخول الرئيسية للموقع
- `js/modules/`: منطق الواجهة، التنقل، البحث، العرض، وربط البيانات
- `css/`: الأنماط البصرية
- `content/reference/`: محتوى المرجع المعياري وصفحات التفاصيل والفهارس
- `data/ui/`: بيانات الواجهة والبحث والتقسيمات
- `.codex/work-progress.json`: ذاكرة تشغيلية لتتبع المرحلة الحالية وما تم إنجازه

## كيف يعمل الموقع؟

عند فتح الموقع:

- تُحمّل الواجهة الأساسية من `index.html`
- تُحمّل ملفات JavaScript المسؤولة عن البحث والتنقل والرندر
- تُقرأ ملفات JSON المحلية حسب القسم المطلوب
- تُعرض الفهارس والصفحات والأمثلة داخل نفس الواجهة

هذا يعني أن المشروع مناسب جدًا للنشر كموقع ثابت على GitHub Pages أو أي استضافة ملفات ساكنة.

## التشغيل محليًا

يمكن تشغيل المشروع محليًا بطريقة بسيطة جدًا. يكفي فتح مجلد المشروع عبر خادم ملفات ثابت.

مثال باستخدام Python:

```bash
python3 -m http.server 8000
```

ثم افتح:

```text
http://localhost:8000
```

ويُفضّل استخدام خادم محلي بدل فتح الملف مباشرة من النظام، لأن بعض المسارات وملفات JSON تعمل بشكل أكثر استقرارًا عبر HTTP.

## النشر كموقع

يمكن نشر المشروع مباشرة عبر `GitHub Pages` لأن بنيته ثابتة ولا تحتاج إلى build server معقد.

المتطلبات المعتادة:

- وجود `index.html` في الجذر
- وجود ملف `.nojekyll` عند الحاجة
- اختيار الفرع المناسب في إعدادات GitHub Pages
- النشر من `/ (root)` إذا كانت الملفات الأساسية موجودة في الجذر

## لمن هذا المشروع؟

هذا المشروع مناسب لـ:

- المبتدئ الذي يريد فهمًا عربيًا أوضح للمفاهيم الرسومية
- المطور الذي يحتاج مرجعًا سريعًا بالعربية
- من يتنقل بين التوثيق الرسمي والأمثلة ويحتاج طبقة شرح تعليمية فوقه
- من يريد استكشاف العلاقة بين المكتبات الرسومية وأدوات البناء والوسائط
